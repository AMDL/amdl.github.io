Section Header

This topic details the AMDL syntax and semantics of section headers.

Function

Section headers serve the following purposes:

  1. Specify the section's title.

  2. Terminate the topic's introduction, if present.

  3. Start a new section.

  4. Optionally, assign values to the section's attributes.

Syntax

Two header syntaxes are defined:

1. setext-Style Section Header:

  1. a line consisting of

    1. zero to three space characters, followed by

    2. text, followed by,

    3. optionally, an attribute enclosure;

  2. a line consisting of

    1. zero to three space characters, followed by,

    2. one or more - characters, followed by,

    3. optionally, any number of space characters.

2. ATX-Style Section Header:

  1. a line consisting of

    1. optionally, up to three space characters, followed by

    2. two to six # characters, followed by

    3. one or more space characters, followed by

    4. text, followed by,

    5. optionally, any number of space characters followed by any number of # characters, followed by,

    6. optionally, an attribute enclosure.

In both syntaxes the text must be XML-escaped, i.e.

  • all occurrences of < must be replaced with &lt;,

  • all occurrences of > must be replaced with &gt;, and

  • all occurrences of & must be replaced with &amp;.

Attributes

Sections may be assigned attributes through attribute enclosures using the following syntax:

{#identifier .class1 .class2 name1="value1" name2="value2" ...}

The following attributes are defined:

identifier

  • a string value used for resolving local links to the section. If no value is specified, a unique identifier will be assigned based on the header text; detailed algorithm TBD.

class

  • Ignored.

autoOutline

  • a Boolean value indicating whether an automatic section outline should be generated. If no attribute value is specified, a section outline should be generated for a section with an setex-style header, and should not be generated for a section with an ATX-style header.

outlineDepth

  • an integer value indicating the maximum depth of the automatic section outline, if one is generated. If no attribute value is specified, a zero-depth outline should be generated.

outlineLead

  • a string value that will be used as the lead-in text rendered before the automatic outline, if one is generated. If an empty string is specified, the lead-in text should be omitted. If no attribute value is specified, a default lead-in text may be used.

noSeeAlso

  • a Boolean value indicating whether the See Also sub-section will be excluded from the automatic section outline, if both are generated. If no attribute value is specified, the See Also sub-section should be excluded if it has an ATX-style header, and shoud not be excluded if it has a setext-style header.

Differences from PM

  1. Class specifications are ignored.
