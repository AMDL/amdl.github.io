ATX-Style Topic Header

This topic details the AMDL syntax and semantics of ATX-style topic headers.

Function

An ATX-style topic header serves the following purposes:

  1. Specify the topic's title.

  2. Signify the termination of the topic's summary, if one is present.

  3. Optionally, assign values to the topic's attributes.

Syntax

A line consisting of:

  1. optionally, up to three space characters, followed by

  2. a # character, followed by

  3. one or more space characters, followed by

  4. unformatted text, followed by,

  5. optionally, any number of space characters followed by any number of # characters, followed by,

  6. optionally, an attribute enclosure.

The unformatted text must be XML-escaped, i.e.

  • all occurrences of < must be replaced with &lt;,

  • all occurrences of > must be replaced with &gt;, and

  • all occurrences of & must be replaced with &amp;.

If a topic's title is unspecified, the topic's identifier will be used.

Example
AMDL
Copy
# ATX-Style Topic Header
Example Output

This topic's header.

Attributes

Topics can be assigned attributes through attribute enclosures using the following syntax:

AMDL
Copy
{#identifier .class1 .class2 name1=value1 name2=value2 ...}

All attribute values containing spaces, braces ({ and }), or backslashes (\) must either

  1. be enclosed in matching ' or " characters, or

  2. have all occurrences of these characters escaped, i.e. preceded by a \ character.

Topic headers define the following attributes:

identifier

  • a globally unique string value used for resolving internal links to the topic. If no value is specified, a unique identifier will be assigned based on the topic file name; detailed algorithm TBD.

tocOrder

  • an integer value that will be used for determining the topic's placement in the Table of Contents.

tocTitle

  • a string value that will be rendered as the link text in the link to the topic in the Table of Contents. If no attribute value is specified, the topic title will be used.

linkText

  • a string value that will be rendered as the link text in internal links to the topic. If no attribute value is specified, the topic title will be used.

autoOutline

  • a boolean value indicating whether an automatic topic outline should be generated. If no attribute value is specified for a topic with an ATX-style header, a topic outline should not be generated.

outlineDepth

  • an integer value indicating the maximum depth of the automatic topic outline, if one is generated. If no attribute value is specified, a zero-depth outline should be generated.

outlineLead

  • a string value that will be used as the lead-in text rendered before the automatic outline, if one is generated. If an empty string is specified, the lead-in text should be omitted. If no attribute value is specified, a default lead-in text may be used.

excludeSeeAlso

  • a boolean value indicating whether the top-level See Also section will be excluded from the automatic topic outline, if both are generated. If no attribute value is specified, the top-level See Also section should not be excluded, i.e. it should be included.

Class specifications should be ignored, but must be accepted as valid input.

Notes

Differences from PM

  1. No inline formatting is permitted in topic headers.

  2. Class specifications are undefined.

amdl2maml Support

Partial, pending parser modification.

  1. Topic titles are only read from the content file.

  2. Outlines are suppressed for all ATX-style headers.

  3. Attributes are misinterpreted as part of the header text.
