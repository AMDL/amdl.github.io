Container

Containers may be used for enclosing topic-specific content.

  • Syntax
  • Classes
  • Differences from PM
  • amdl2maml Support
Syntax

  1. a line consisting of

    1. n : (colon) characters, where n ≥ 3, followed by

    2. a class attribute value;

  2. content consisting of any number of lines, where each line

    1. does not start with l : characters where l ≥ n;

  3. a line starting with m : characters where m ≥ n.

If no class is specified, or if the specified class is unsupported by the specific topic type, the container will be interpreted as an alert.

 Note
The class attribute is case-sensitive.
Classes

The classes listed below are supported by all topic types. Any classes supported by a specific topic type in addition to these are listed in the corresponding Topic Types topic.

  1. relatedTopics - equivalent to a See Also section.

  2. concepts - equivalent to a Concepts sub-section in a See Also section.

  3. otherResources - equivalent to a Other Resources sub-section in a See Also section.

  4. reference - equivalent to a Reference sub-section in a See Also section.

  5. tasks - equivalent to a Tasks sub-section in a See Also section.

Differences from PM

  1. No container elements exist in PM.

amdl2maml Support

Partial, pending parser modification.

Limitations

  1. Containers can only contain inline elements.
