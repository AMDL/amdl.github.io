Setext-Style Section Header

This topic details the AMDL syntax and semantics of Setext-style section headers.

Function

A Setext-style section header serves the following purposes:

  1. Specify the section's title.

  2. Terminate the topic's introduction, if present.

  3. Start a new section.

  4. Optionally, assign values to the section's attributes.

Syntax

  1. A line consisting of:

    1. zero to three space characters, followed by

    2. text, followed by,

    3. optionally, an attribute enclosure, followed by

  2. a line consisting of:

    1. zero to three space characters, followed by,

    2. at least three - character, followed by,

    3. optionally, any character sequence.

The text must be XML-escaped, i.e.

  • all occurrences of < must be replaced with &lt;,

  • all occurrences of > must be replaced with &gt;, and

  • all occurrences of & must be replaced with &amp;.

Notes

Example Output

The Notes section.

Attributes

Sections can be assigned attributes through attribute enclosures using the following syntax:

AMDL
Copy
{#identifier .class1 .class2 name1=value1 name2=value2 ...}

All attribute values containing spaces, braces ({ and }), or backslashes (\) must either

  1. be enclosed in matching ' or " characters, or

  2. have all occurrences of these characters escaped, i.e. preceded by a \ character.

Section headers define the following attributes:

identifier

  • a string value used for resolving internal links to the section. If no value is specified, a unique identifier will be assigned based on the header text; detailed algorithm TBD.

autoOutline

  • a boolean value indicating whether an automatic section outline should be generated. If no attribute value is specified for a section with an Setex-style header, a section outline should be generated.

outlineDepth

  • an integer value indicating the maximum depth of the automatic section outline, if one is generated. If no attribute value is specified, a zero-depth outline should be generated.

outlineLead

  • a string value that will be used as the lead-in text rendered before the automatic outline, if one is generated. If an empty string is specified, the lead-in text should be omitted. If no attribute value is specified, a default lead-in text may be used.

excludeSeeAlso

  • a boolean value indicating whether the See Also sub-section will be excluded from the automatic section outline, if both are generated. If no attribute value is specified, the See Also sub-section should not be excluded, i.e. it should be included.

Class specifications should be ignored, but must be accepted as valid input.
